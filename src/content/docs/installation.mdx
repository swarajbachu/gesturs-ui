<Steps>

<Step>Create project</Step>

Start by creating a new Next.js project using `create-next-app`:

```bash
npx create-next-app@latest my-app --typescript --tailwind --eslint
```

<Step>Install dependencies</Step>

```bash
npm install react-aria-components tailwindcss-react-aria-components tailwindcss-animate class-variance-authority clsx tailwind-merge
```

<Step>Add icon library</Step>

```bash
npm install lucide-react
```

<Step>Configure path aliases</Step>

I use the `@` alias. This is how I configure it in tsconfig.json:

```json {3-5}
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

The `@` alias is a preference. You can use other aliases if you want.

**If you use a different alias such as `~`, you'll need to update import statements when adding components.**

<Step>Add a cn helper</Step>

I use a cn helper to make it easier to conditionally add Tailwind CSS classes. Here's how I define it in `lib/utils.ts`:

```tsx
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```
